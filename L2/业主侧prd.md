# 业主方项目治理与风险驾驶舱 PRD（V1.0）
## 1. 产品背景与定位
### 1.1 背景
在政务平台建设、信息化项目、重点专项等场景中，传统项目管理系统以项目经理视角为主，侧重任务、进度与资源管理，难以满足业主方（甲方、监管方、领导层）对以下问题的核心诉求：
* 项目是否存在重大风险？
* 是否符合规范与审计要求？
* 是否有人负责并形成闭环？
* 是否可能留下治理与问责隐患？

### 1.2 产品定位
本系统定位为：
**以业主方视角为核心的项目治理与风险驾驶舱系统**。
核心不是“管理项目怎么干”，而是帮助业主方判断：
* 是否安全
* 是否可靠
* 是否可追责
* 是否可审计

### 1.3 核心价值
* 提前发现风险
* 强化履约与合规
* 明确责任与闭环
* 沉淀过程资产
* 支撑审计与问责


## 2. 目标用户与使用场景
### 2.1 目标用户
* 主管领导 / 分管领导
* 业主单位项目主管
* 监管与审计人员
* 平台建设统筹部门

### 2.2 核心使用场景
* 领导快速判断项目是否需要介入
* 业主方例会 / 汇报查看项目风险
* 审计、巡查、绩效考核支撑
* 项目验收与后评估决策支持


## 3. 产品设计原则（业主方视角）
* 风险优先，而非进度优先
* 异常优先，而非任务细节
* 责任优先，而非人员列表
* 治理证据优先，而非过程描述
* 自动总结优先，而非人工解读

## 4. 业主方核心判断模型
系统需支持业主方在 10 秒内回答以下 4 个核心问题：
1. 现在安全吗？（风险）
2. 交付靠谱吗？（质量）
3. 有没有人兜底？（责任）
4. 以后会不会留下坑？（合规与审计）


## 5. 业主方单项目驾驶舱总体结构
### 第一层：总体态势（3 秒判断）
* 项目状态灯（绿 / 黄 / 红）
* 项目治理健康指数
* 是否建议领导介入（是 / 否）
* 红色预警数量

### 第二层：四大治理维度（10 秒定位问题）
* 风险
* 质量
* 责任
* 合规 / 治理

### 第三层：介入依据（给领导的理由）
* 异常画像自动总结
* 最近关键异常
* 最近领导督办
* 风险趋势说明


## 6. 核心功能模块
### 6.1 风险治理模块
#### 目标
让业主快速识别是否存在可能引发重大后果的风险。

#### 核心指标
* 未关闭风险数量
* 风险关闭率

#### 功能
* 风险超期自动升级
* 风险趋势（上升 / 下降）


### 6.2 交付质量与验收模块

#### 目标

评估项目是否稳定交付，是否存在反复整改或质量不稳定。

#### 核心指标

* 一次验收通过率
* 平均整改次数
* 未关闭质量问题数
* 重大质量问题数

#### 功能
* 多次整改自动预警
* 重大质量问题自动标红
* 验收稳定性趋势
