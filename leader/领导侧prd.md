一、单项目状态机（Lifecycle + 状态流转）
1. 立项与决策
- 项目申报
- 可研/论证中
- 立项评审中
- 立项通过
- 立项驳回
- 项目终止
2. 启动与规划
- 项目启动
- 项目负责人确认
- 项目计划编制中
- 预算确认中
- 启动完成
3. 执行与控制（核心）
- 正常执行
- 里程碑执行中
- 阶段验收中
- 风险处理中
- 变更审批中
- 进度异常 ⚠️
- 成本异常 ⚠️
- 质量异常 ⚠️
支持并行异常标签（一个项目可同时存在多个异常）
4. 收尾与验收
- 完工待验收
- 验收中
- 验收未通过（整改中）
- 验收通过
5. 后评估与关闭
- 投入使用
- 后评估中
- 后评估完成
- 项目关闭归档

二、单项目详情页（领导视角）页面结构
模块 1：项目总体态势（首页第一屏）
字段：
- 项目健康指数（0–100）
- 项目状态灯（绿/黄/红）
- 计划进度（%）
- 实际进度（%）
- 进度偏差（天 / %）
- 总预算
- 已使用资金
- 资金使用率
- 高风险数量
- 未关闭风险数量
规则：
- 超过阈值自动变黄/红
- 第一屏不超过 8–10 个核心指标

模块 2：项目时间轴 / 关键里程碑
字段：
- 里程碑名称
- 计划开始/结束时间
- 实际开始/结束时间
- 当前状态（未开始/进行中/已完成/延期）
- 延期天数
能力：
- 当前节点高亮
- 延期节点自动标红

模块 3：任务与责任体系
字段：
- 项目负责人
- 分管领导
- 承办单位
- 协作单位
- 当前关键任务
- 逾期任务数
能力：
- 任务逾期自动预警
- 支持责任到人

模块 4：风险、问题、变更（异常区）
风险管理
- 风险名称
- 风险等级（高/中/低）
- 风险类型（进度/资金/政策/技术/资源等）
- 应对措施
- 责任人
- 当前状态（未处理/处理中/已关闭）
问题管理
- 问题来源（验收/审计/巡查/监管发现等）
- 问题等级
- 整改要求
- 整改截止时间
- 整改状态
变更管理
- 变更类型（范围/工期/资金/目标）
- 变更原因
- 对工期影响（+天）
- 对资金影响（+金额）
- 审批状态


模块 5：验收与成果
字段：
- 阶段验收记录
- 验收结论
- 整改意见
- 整改次数
- 交付成果清单
- 验收通过率

模块 6：资金与合同
字段：
- 合同数量
- 合同总金额
- 已支付金额
- 待支付金额
- 付款节点
- 资金风险提示

七、审批流与状态联动设计（核心流程）
1. 立项审批流
- 项目申报 → 部门初审 → 专家评审 → 主管领导审批 → 立项通过/退回
联动：
- 通过后自动进入【启动与规划】阶段
- 退回自动生成整改任务

2. 变更审批流
- 提交变更 → 项目负责人审核 → 业务主管审批 → 财务审核（涉及资金） → 领导审批
联动：
- 重大变更自动降低健康指数
- 变更通过后自动更新计划/预算基线

3. 验收审批流
- 提交验收申请 → 组织验收 → 形成验收结论 → 验收通过/整改
联动：
- 验收未通过自动生成问题与整改任务
- 多次未通过自动触发黄色/红色预警

八、预警规则引擎设计（监管系统灵魂）
预警类型
- 进度预警
- 成本预警
- 风险预警
- 验收预警
- 督办预警
核心规则示例
- 里程碑延期 ≥ 7 天 → 黄色预警
- 里程碑延期 ≥ 15 天 → 红色预警
- 资金使用率 ≥ 90% → 黄色预警
- 资金使用率 ≥ 100% → 红色预警
- 高风险未关闭 ≥ 2 个 → 红色预警
- 督办事项超期 → 红色预警
能力：
- 规则可配置
- 支持按项目类型差异化配置

九、角色与权限模型（领导监管视角）
角色：
- 系统管理员
- 主管领导
- 分管领导
- 项目负责人
- 承办单位人员
- 审计/监管人员
- 财务人员
权限要点：
- 领导：只读 + 批示 + 督办
- 项目负责人：编辑项目、提交变更、提交验收
- 审计/监管：查看全部 + 提交问题
- 财务：资金与合同模块编辑
