三、单项目健康评分模型（示例）
评分维度（总分 100）：
1. 进度（30分）
- 无延期：30
- 延期 ≤ 7天：20
- 延期 > 7天：10
- 严重延期：0
2. 成本（25分）
- 未超预算：25
- 接近预算（>90%）：15
- 超预算：5
3. 风险（20分）
- 无高风险：20
- 1个高风险：10
- ≥2个高风险：0
4. 质量/问题（15分）
- 无重大问题：15
- 有未整改问题：8
- 多次整改：0
5. 变更稳定性（10分）
- 无重大变更：10
- 有重大变更：5
- 多次重大变更：0
输出：
- 健康指数 = 五项得分之和
- 状态灯：
  - ≥80：绿色
  - 60–79：黄色
  - <60：红色

八、审批流与状态联动设计（核心流程）
1. 立项审批流
- 项目申报 → 部门初审 → 专家评审 → 主管领导审批 → 立项通过/退回
联动：
- 通过后自动进入【启动与规划】阶段
- 退回自动生成整改任务

---
2. 变更审批流
- 提交变更 → 项目负责人审核 → 业务主管审批 → 财务审核（涉及资金） → 领导审批
联动：
- 重大变更自动降低健康指数
- 变更通过后自动更新计划/预算基线

---
3. 验收审批流
- 提交验收申请 → 组织验收 → 形成验收结论 → 验收通过/整改
联动：
- 验收未通过自动生成问题与整改任务
- 多次未通过自动触发黄色/红色预警
